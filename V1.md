# review-gate v1.0 — the cut line

A multi-model PR review **gate**: untrusted model reviewers + a trusted **deterministic spine** that
owns the verdict and the trust boundary. This file is the v1 definition — what's in, what's deferred —
so "done" is a fixed line, not a moving target.

## What v1.0 is (built, merged, tested)

- **Deterministic spine** (`consolidate` + `decide`): clusters findings by location, computes
  cross-model agreement, enforces **no-silent-dismissal**, emits one `block`/`pass` verdict + a single
  PR comment. A prompt-injected diff or steered agent cannot flip the gate, bury a finding, or dismiss
  a tool-detected fact.
- **Orchestrator skill** (`review-gate`): holistic-first review + conditional lenses, the
  signing-authority mandate, the mandatory written lens decision.
- **Deterministic scan tier**: git-hygiene (always); secrets/deps (gitleaks / osv-scanner when
  installed) — **fail-closed** (a scanner that can't complete blocks, never silently passes).
- **Installable Claude Code plugin**: the two skills (`review-gate`, `repo-audit`) + a `review-gate`
  CLI on `PATH` that serves its own prompts. Committed `dist/` → no install-time build.
- **Reviewer infra hardened**: each reviewer run is bounded by `--max-turns` (cost circuit-breaker)
  and a hard wall-clock timeout that **force-settles and tears down stdio** — so a non-converging or
  hung model can neither wedge the gate nor run away on Ollama Cloud's GPU-time billing. Proven by
  reproduction tests (orphaned-pipe hang; host-process exit; byte-cap).

**Reliable reviewer panel:** `kimi-k2.7-code` (ollama) + `gpt-5.5` (codex) parse cleanly every run.
`claude-opus-4-8` and `glm-5.2` contribute **best-effort** — they sometimes return prose instead of a
JSON array; the runner salvages a fenced array when one is present, otherwise that reviewer is a
surfaced non-vote.

**Verified:** 119 tests (no network) · `tsc --noEmit` clean · `build:check` (committed dist == src).

## Install
```
/plugin marketplace add smarzban/review-gate
/plugin install review-gate@smarzban
```
Prereqs (user-installed): `node`, `gh`, `git`, ≥1 model backend (ollama / claude / codex);
optional `gitleaks` + `osv-scanner` for the full scan tier.

## How it got here
Distilled from a bake-off (a clean holistic prompt across a few diverse models beat a 30-reviewer
specialist panel at far lower cost; the durable value was the deterministic spine, not the
specialization). Then hardened through **adversarial self-dogfooding** across PRs #1–5, which caught
real fail-opens — argument injection, an embedded-`[]` clean-pass forge, a comma-hedge, a unicode
bypass, an adjudication-key collision — and a cost-runaway (an orphaned process that outlived the
timeout). Each was reproduced and fixed under test. The gate works because it was made to review
itself.
